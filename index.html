What changed

The app now uses a verified snapshot layer first, so it can build a workable show even before any browser-side refresh succeeds. That makes it more dependable because your notes, probable-pitcher references, standings framing, and transaction pegs are already structured when the page opens. [web:50][web:188][web:279]

Guardrail

Roster and injury notes still need verification, so the app is designed to surface transactions from MLB.com or ESPN lanes before they become script notes. If a move cannot be confirmed from available fetched or embedded sources, do not use it on-air. [web:250][web:194][web:263][web:261][web:279]
